제공해주신 『쉽게 배우는 운영체제』와 『컴퓨터 시스템 구조』의 내용을 바탕으로, 기술 면접 대비용 CS/CE(Computer Science/Computer Engineering) 핵심 질문 10가지를 선정해 드립니다. 각 질문은 실제 면접에서 자주 등장하는 주제들입니다.

### 1. 프로세스(Process)와 스레드(Thread)의 차이점은 무엇인가요?

- **출제 의도:** 운영체제의 가장 기본적인 실행 단위에 대한 이해도를 묻습니다.
- **핵심 키워드:** 자원 공유, 독립성, 문맥 교환(Context Switching)
- **참고 내용:** 프로세스는 실행 중인 프로그램으로 메모리에 적재되어 CPU의 할당을 받는 작업의 단위입니다. 반면 스레드는 프로세스 내에서 실행되는 흐름의 단위로, 프로세스의 자원(코드, 데이터 등)을 공유합니다. 멀티태스킹과 멀티스레드의 차이점, 그리고 스레드 간 통신이 프로세스 간 통신(IPC)보다 효율적인 이유 등을 설명할 수 있어야 합니다.

### 2. 컨텍스트 스위칭(Context Switching)이란 무엇이며, 언제 발생하나요?

- **출제 의도:** CPU가 여러 프로세스를 번갈아 처리하는 매커니즘을 이해하는지 확인합니다.
- **핵심 키워드:** PCB(Process Control Block), 오버헤드, 인터럽트, 타임 슬라이스
- **참고 내용:** CPU를 차지하던 프로세스가 나가고 새로운 프로세스를 받아들이는 작업입니다. 실행 중인 프로세스의 상태(문맥)를 PCB에 저장하고, 새로운 프로세스의 상태를 PCB에서 읽어와 레지스터에 적재하는 과정입니다. 주로 할당된 시간(타임 슬라이스)이 끝나거나, I/O 요청, 인터럽트 발생 시 일어납니다.

### 3. 교착 상태(Deadlock)가 무엇이며, 발생하기 위한 4가지 필요조건은 무엇인가요?

- **출제 의도:** 시스템 자원 관리 중 발생할 수 있는 치명적인 문제와 그 원인을 아는지 묻습니다.
- **핵심 키워드:** 상호 배제, 비선점, 점유와 대기, 원형 대기
- **참고 내용:** 2개 이상의 프로세스가 서로 상대방의 작업이 끝나기만을 기다리며 멈춰있는 상태입니다. 발생 조건은 ①상호 배제(한 번에 하나만 사용), ②비선점(빼앗을 수 없음), ③점유와 대기(자원을 가진 채 다른 자원 대기), ④원형 대기(대기 관계가 원을 이룸)이며, 이 중 하나라도 만족하지 않으면 교착 상태는 발생하지 않습니다.

### 4. 가상 메모리(Virtual Memory) 관리 기법인 페이징(Paging)과 세그먼테이션(Segmentation)의 차이는 무엇인가요?

- **출제 의도:** 메모리 관리 및 단편화(Fragmentation) 문제에 대한 이해를 묻습니다.
- **핵심 키워드:** 고정 분할, 가변 분할, 내부 단편화, 외부 단편화
- **참고 내용:** 페이징은 물리 메모리를 고정된 크기(프레임)로 나누고 논리 메모리도 같은 크기(페이지)로 잘라 매핑하는 방식으로, 외부 단편화는 없으나 내부 단편화가 발생할 수 있습니다. 세그먼테이션은 논리적 단위(함수, 배열 등)대로 가변적인 크기로 자르는 방식으로, 내부 단편화는 없으나 외부 단편화가 발생할 수 있습니다.

### 5. 페이지 교체 알고리즘 중 LRU(Least Recently Used)와 FIFO의 차이를 설명해주세요.

- **출제 의도:** 메모리가 부족할 때 어떤 페이지를 내보낼지 결정하는 알고리즘에 대한 지식을 묻습니다.
- **핵심 키워드:** 시간 지역성, 큐(Queue), 오버헤드
- **참고 내용:** FIFO는 메모리에 가장 먼저 들어온 페이지를 교체하는 단순한 방식이나 성능이 좋지 않을 수 있습니다. LRU는 가장 오랫동안 사용되지 않은 페이지를 교체하는 방식으로, 시간 지역성(최근에 사용된 데이터가 다시 사용될 확률이 높음)을 고려하여 성능이 더 우수하지만 구현이 복잡할 수 있습니다.

### 6. RISC(Reduced Instruction Set Computer)와 CISC(Complex Instruction Set Computer) 아키텍처의 차이점은 무엇인가요?

- **출제 의도:** 컴퓨터 프로세서 설계 철학의 차이를 이해하는지 묻습니다.
- **핵심 키워드:** 명령어 수, 명령어 길이, 파이프라인, 레지스터
- **참고 내용:** CISC는 하나의 명령어로 복잡한 작업을 수행하며 명령어 길이가 가변적이고 종류가 많습니다(예: Intel x86). RISC는 간단하고 자주 쓰이는 명령어 위주로 구성되며, 명령어 길이가 고정적이고 파이프라이닝에 최적화되어 있어 실행 속도가 빠릅니다(예: ARM).

### 7. 파이프라인(Pipeline) 기법이란 무엇이며, 파이프라인 해저드(Hazard)는 왜 발생하나요?

- **출제 의도:** CPU 성능 향상 기법과 그 한계점에 대해 묻습니다.
- **핵심 키워드:** 병렬 처리, 데이터 의존성, 분기(Branch)
- **참고 내용:** 파이프라인은 하나의 명령어가 실행되는 도중에 다음 명령어를 미리 가져와 실행 단계들을 중첩시켜 처리 속도를 높이는 기법입니다. 해저드는 데이터 의존성(앞 명령어의 결과가 필요함), 제어 의존성(분기문 등), 구조적 문제 등으로 인해 파이프라인이 멈추거나 지연되는 현상을 말합니다.

### 8. 캐시 메모리(Cache Memory)의 역할과 지역성(Locality)의 원리에 대해 설명해주세요.

- **출제 의도:** 메모리 계층 구조와 속도 차이를 극복하는 원리를 아는지 묻습니다.
- **핵심 키워드:** 시간적 지역성, 공간적 지역성, 적중률(Hit Ratio)
- **참고 내용:** 캐시는 CPU와 주기억장치 사이의 속도 차이를 완화하기 위한 고속 메모리입니다. 지역성 원리를 이용하는데, 시간적 지역성은 한 번 참조된 데이터가 곧 다시 참조될 가능성을, 공간적 지역성은 참조된 데이터 인근의 데이터가 참조될 가능성을 의미합니다.

### 9. DMA(Direct Memory Access)란 무엇이며 왜 필요한가요?

- **출제 의도:** 입출력 처리 효율성과 CPU의 부하 감소에 대한 이해를 묻습니다.
- **핵심 키워드:** CPU 개입 최소화, 버스 공유, 사이클 스틸링(Cycle Stealing)
- **참고 내용:** 입출력 장치가 CPU를 거치지 않고 직접 메모리에 접근하여 데이터를 전송하는 방식입니다. 대량의 데이터 이동 시 CPU의 부하를 줄여 시스템 효율을 높입니다. DMA가 메모리 버스를 사용할 때 CPU가 잠시 기다리는 것을 사이클 스틸링이라고 합니다.

### 10. 인터럽트(Interrupt)의 처리 과정을 설명해주세요.

- **출제 의도:** 하드웨어/소프트웨어 이벤트 발생 시 운영체제의 제어 흐름을 묻습니다.
- **핵심 키워드:** 인터럽트 벡터, ISR(인터럽트 서비스 루틴), 문맥 저장
- **참고 내용:** 입출력 완료, 예외 발생 등의 이벤트가 생기면 CPU는 하던 일을 멈추고 현재 상태(PC 등)를 저장합니다. 그 후 인터럽트 벡터를 통해 해당 이벤트를 처리할 함수(인터럽트 핸들러/서비스 루틴)의 주소를 찾아 실행하고, 처리가 끝나면 원래 작업으로 복귀합니다.
