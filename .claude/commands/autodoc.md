# /autodoc - 자동 문서 작성 시스템

## 개요
주어진 주제에 대해 자동으로 문서를 작성하고 퇴고하는 명령어입니다.

**7단계 자동 작성 & 퇴고 흐름:**
1. **Step 1**: 주제에 필요한 소제목 & 용어 수집
2. **Step 2**: 1차 검토 (과한 수준 내용 체크)
3. **Step 3**: 초안 작성 (서술식)
4. **Step 4**: 2차 정제 (과도 내용 제거)
5. **Step 5**: 마크다운 문법 검수
6. **Step 6**: 코드 예시 검증
7. **Step 7**: 환각 & 창작 내용 검증

## 사용법

```bash
/autodoc
```

명령 실행 후:
1. "어떤 주제의 문서를 작성하고 싶으신가요?"라는 질문에 **주제를 입력**
2. 자동으로 4단계 흐름이 진행됨

## 사용 예시

```
User: /autodoc
Assistant: 어떤 주제의 문서를 작성하고 싶으신가요?

User: 스마트포인터에 대해서
Assistant: [Step 1 진행...]
          [Step 2 진행...]
          [Step 3 진행...]
          [Step 4 진행...]
          [Step 5 마크다운 문법 검수...]
          [Step 6 코드 예시 검증...]
          [Step 7 환각 & 창작 내용 검증...]
          ✅ 완료!
```

## 각 단계 상세 설명

### Step 5: 마크다운 문법 검수
**검사 항목:**
- 코드 블록 문법 (```)
- 헤더 레벨 (# ## ###)
- 링크 및 이미지 문법
- 표(Table) 형식
- 강조 문법 (**bold**, *italic*)
- 목록 들여쓰기
- 특수 문자 이스케이핑

**수정 작업:**
- 잘못된 문법 즉시 수정
- 일관성 없는 형식 통일

### Step 6: 코드 예시 검증
**검사 항목:**
- 문법 오류 (컴파일 불가 코드)
- 변수명 오타
- 함수 시그니처 정확성
- 타입 불일치
- 메모리 관리 실수
- 언어별 특화 내용 (C++, 언리얼)
- 주석과 코드 일관성

**수정 작업:**
- 잘못된 코드 수정
- 더 나은 예시로 대체
- 설명 추가

### Step 7: 환각 & 창작 내용 검증
**검사 항목:**
- 실제 존재하지 않는 함수/클래스
- 잘못된 버전 정보 (예: "C++98에서 unique_ptr 도입" - 거짓)
- 사실이 아닌 역사적 정보
- 존재하지 않는 언리얼 엔진 기능
- 부정확한 성능 수치
- 잘못된 개념 설명
- 창작으로 만들어진 API

**수정 작업:**
- 정확한 정보로 교체
- 불확실한 내용 제거
- 검증 가능한 사실만 유지

## 작성 특징

✅ **서술식 작성**
- 단순한 용어 정의가 아닌 이야기하듯 설명
- 개념이 필요한 이유부터 설명

✅ **맥락에 맞는 서술 방향**
- 문제→해결책 흐름 (필요시)
- 단순 설명 (주제에 맞게)

✅ **언리얼 엔진 최적화**
- 게임 개발 예시
- 언리얼 특화 내용

✅ **정제된 콘텐츠**
- 과도하게 깊은 내용 제거
- 대상 독자 수준에 맞게 조정

## 생성 파일 위치

작성된 문서는 `Document/` 폴더 하위에 저장됩니다.

예: `Document/SmartPointer.md`, `Document/Socket.md`

## 퇴고 단계 실행 방식

**Step 5-7은 문서 작성 후 자동으로 실행됩니다:**

1. 작성된 문서 전체를 대상으로 마크다운 검수
2. 각 코드 예시를 문법/논리 검증
3. 모든 기술적 내용과 API 검증
4. 발견된 오류 목록 작성 및 수정

## 주의사항

- 주제를 명확하게 입력해야 정확한 문서가 생성됩니다
- Step 2에서 구성 변경 가능 (진행 중단 후 수정 가능)
- Step 4 이후 추가 수정 사항이 있으면 언제든 요청 가능
- **Step 5-7의 퇴고는 자동 실행되므로 별도 요청 불필요**

## 퇴고 결과 보고

각 단계별로 발견된 오류와 수정 사항을 명확히 보고합니다:

```
✅ Step 5 마크다운 문법 검수 완료
   - 수정: 3곳의 코드 블록 들여쓰기
   - 수정: 표 포맷 2개 통일

✅ Step 6 코드 예시 검증 완료
   - 수정: shared_ptr 예시의 타입 불일치 (1개)
   - 확인: 모든 C++ 문법 정확

✅ Step 7 환각 & 창작 내용 검증 완료
   - 확인: 모든 API 실제 존재
   - 확인: 모든 기술 정보 정확
```
